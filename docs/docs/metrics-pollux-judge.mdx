The Pollux Judge metric evaluates `actual_output` using the
[POLLUX](https://huggingface.co/collections/ai-forever/pollux)
rubric-based LLM judge.

Unlike built-in fixed-criteria metrics, `PolluxJudgeMetric` accepts a
custom criterion (`criteria_name`) and custom rubric scale (`rubrics`)
at initialization time.

## Required Arguments

To use `PolluxJudgeMetric`, provide these fields in
[`LLMTestCase`](/docs/evaluation-test-cases#llm-test-case):

- `input`
- `actual_output`
- `expected_output` (optional, used as reference answer if present)

## Run Pollux Locally

Before using `PolluxJudgeMetric`, start a local POLLUX judge endpoint
with an OpenAI-compatible API (for example, vLLM):

```bash
pip install vllm
vllm serve ai-forever/Pollux-4B-Judge --port 8000
```

Then point the metric to your local endpoint:
- `base_url="http://localhost:8000/v1"`
- `api_key="NONE"` for local unauthenticated serving

## Usage

```python
from deepeval import evaluate
from deepeval.metrics import PolluxJudgeMetric
from deepeval.test_case import LLMTestCase

metric = PolluxJudgeMetric(
    criteria_name="Correctness",
    rubrics={
        0: "Wrong answer or no answer.",
        1: "Partially correct answer.",
        2: "Fully correct answer.",
    },
    judge_model="ai-forever/Pollux-4B-Judge",
    base_url="http://localhost:8000/v1",  # OpenAI-compatible endpoint
    api_key="NONE",
    threshold=0.5,
    normalize_score=True,
    strict_mode=False,
    async_mode=True,
)

test_case = LLMTestCase(
    input="What is 2 + 2?",
    actual_output="The answer is 4.",
    expected_output="4",
)

evaluate(test_cases=[test_case], metrics=[metric])
```

## Parameters

- `criteria_name` (required): Name of the evaluation criterion.
- `rubrics` (required): Dictionary score-to-description, for example
  `{0: "...", 1: "...", 2: "..."}`.
- `judge_model` (optional): Judge model id, default
  `ai-forever/Pollux-4B-Judge`.
- `base_url` (optional): OpenAI-compatible endpoint URL, default
  `http://localhost:8000/v1`.
- `api_key` (optional): API key for the endpoint, default `"NONE"`.
- `max_tokens` (optional): Max completion tokens from the judge.
- `temperature` (optional): Judge sampling temperature.
- `threshold` (optional): Passing threshold when `strict_mode=False`.
- `normalize_score` (optional): If `True`, maps rubric scores to
  `[0.0, 1.0]`.
- `include_reason` (optional): If `True`, includes parsed feedback in
  `metric.reason`.
- `strict_mode` (optional): If `True`, passing threshold is forced to
  perfect-score threshold:
  - `1.0` when `normalize_score=True`
  - `max(rubric_keys)` when `normalize_score=False`
- `async_mode` (optional): If `True`, `measure()` runs through
  `a_measure()`.
- `verbose_mode` (optional): If `True`, stores intermediate logs in
  `metric.verbose_logs`.
- `score_pattern` (optional): Regex with one capture group for the numeric
  score. Default expects a **plain numeric** response (optional whitespace),
  e.g. `2` or `\n\n2` (same as `POLLUX_DEFAULT_SCORE_RE` in `deepeval.metrics.pollux`).
  For judges that return tagged output, pass
  `POLLUX_TAGGED_SCORE_RE` from `deepeval.metrics.pollux`.
- `feedback_pattern` (optional): Regex with one capture group for feedback.
  Default ``None`` — no feedback (`metric.reason` is empty when `include_reason=True`; same semantics as :data:`POLLUX_DEFAULT_FEEDBACK_RE`).
  For tagged judges, pass `POLLUX_TAGGED_FEEDBACK_RE` from `deepeval.metrics.pollux`.

### Tagged judge output (legacy format)

If your judge returns `[FEEDBACK] ... [RESULT] <score> [END]`, use:

```python
from deepeval.metrics.pollux import (
    PolluxJudgeMetric,
    POLLUX_TAGGED_FEEDBACK_RE,
    POLLUX_TAGGED_SCORE_RE,
)

metric = PolluxJudgeMetric(
    criteria_name="Correctness",
    rubrics={0: "Wrong", 1: "Partial", 2: "Correct"},
    score_pattern=POLLUX_TAGGED_SCORE_RE,
    feedback_pattern=POLLUX_TAGGED_FEEDBACK_RE,
)
```

## As a Standalone

```python
metric.measure(test_case)
print(metric.score, metric.reason, metric.success)
```

## How Is It Calculated?

1. Build POLLUX prompt from:
   - test case `input`
   - test case `actual_output`
   - optional `expected_output`
   - `criteria_name`
   - normalized rubric text
2. Send prompt to OpenAI-compatible chat completions API.
3. Parse score using `score_pattern` (default: entire response is one number).
4. Parse feedback using `feedback_pattern` (default: no feedback).
5. Set final score:
   - normalized score `(score - min_key) / (max_key - min_key)` if
     `normalize_score=True` (clamped to `[0.0, 1.0]`)
   - raw parsed score if `normalize_score=False`

The metric passes when `score >= threshold`.
